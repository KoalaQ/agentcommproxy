## 三、Linux 部署

### 3.1 快速安装（推荐）

将 `agentcommproxy-1.0.0.jar` 和 `install.sh` 复制到目标机器同一目录：

```bash
# 1. 复制文件
scp target/agentcommproxy-1.0.0.jar user@server:/tmp/
scp install.sh user@server:/tmp/
scp daemon.sh user@server:/tmp/

# 2. 登录服务器执行安装
ssh user@server
cd /tmp
sudo ./install.sh
```

### 3.2 验证安装

```bash
agentproxy --help
agentproxy http status
```

### 3.3 启动服务

**方式一：使用 daemon.sh（后台运行）**

```bash
# 启动
/opt/agentproxy/daemon.sh start

# 查看状态
/opt/agentproxy/daemon.sh status

# 停止
/opt/agentproxy/daemon.sh stop

# 重启
/opt/agentproxy/daemon.sh restart
```

**方式二：使用 CLI 命令（前台运行）**

```bash
# 前台运行（阻塞）
agentproxy daemon start --foreground

# 使用 nohup 后台运行（使用配置文件中的端口）
nohup agentproxy daemon start --foreground > ~/.agentcommproxy/logs/daemon.log 2>&1 &

# 指定 HTTP 端口后台运行（全局选项需放在 daemon 子命令之前）
nohup agentproxy --http-port 9091 daemon start --foreground > ~/.agentcommproxy/logs/daemon.log 2>&1 &
```

> **注意：** 全局选项 `--http-port` 必须放在子命令 `daemon` 之前，否则不会被正确解析。

### 3.4 systemd 服务配置（可选）

创建 systemd 服务实现开机自启：

```bash
sudo vi /etc/systemd/system/agentproxy.service
```

内容：

```ini
[Unit]
Description=AgentCommProxy Daemon
After=network.target

[Service]
Type=simple
User=<your-user>
ExecStart=/usr/local/bin/agentproxy daemon start --foreground
Restart=always
RestartSec=10
StandardOutput=append:%h/.agentcommproxy/logs/daemon.log
StandardError=append:%h/.agentcommproxy/logs/daemon.log

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable agentproxy
sudo systemctl start agentproxy
sudo systemctl status agentproxy
```

### 3.5 查看日志

```bash
tail -f ~/.agentcommproxy/logs/daemon.log
```

## 四、Windows 部署

### 4.1 快速安装

将 `agentcommproxy-1.0.0.jar` 和 `install.bat` 复制到同一目录，右键 `install.bat` 选择"以管理员身份运行"。

### 4.2 验证安装

重新打开命令提示符：

```cmd
agentproxy --help
agentproxy http status
```

### 4.3 启动服务

**前台运行：**

```cmd
agentproxy daemon start --foreground
```

**后台运行（使用 PowerShell）：**

```powershell
Start-Process -NoNewWindow java -ArgumentList "-jar", "C:\Program Files\agentproxy\agentcommproxy-1.0.0.jar", "daemon", "start", "--foreground"
```

### 4.4 查看日志

```cmd
type %USERPROFILE%\.agentcommproxy\logs\daemon.log
```

## 五、Docker 部署

### 5.1 构建 Docker 镜像

创建 `Dockerfile`：

```dockerfile
FROM eclipse-temurin:17-jre

WORKDIR /app

COPY agentcommproxy-1.0.0.jar /app/

# 创建数据目录
RUN mkdir -p /root/.agentcommproxy/logs

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "/app/agentcommproxy-1.0.0.jar"]
CMD ["daemon", "start", "--foreground"]
```

构建镜像：

```bash
docker build -t agentcommproxy:1.0.0 .
```

### 5.2 运行容器

```bash
docker run -d \
  --name agentproxy \
  -p 8080:8080 \
  -v ~/.agentcommproxy:/root/.agentcommproxy \
  agentcommproxy:1.0.0
```

### 5.3 查看日志

```bash
docker logs -f agentproxy
```

## 六、配置说明

### 6.1 配置文件位置

配置文件：`~/.agentcommproxy/config.properties`

首次启动自动生成，默认配置：

```properties
# 默认超时时间（秒）
default.timeout=300

# 异步重试次数
async.retry.count=3

# 重试间隔（秒）
async.retry.interval=5

# 守护进程配置
daemon.enabled=true
daemon.thread.pool.size=4
daemon.scan.interval=5

# HTTP 服务配置
http.enabled=true
http.port=8080
http.api.key=<auto-generated-uuid>

# Proxy 配置
proxy.default=openclaw

# 定期清理配置
cleanup.enabled=true
cleanup.days=7
cleanup.status=DONE
```

### 6.2 常用配置修改

**修改 HTTP 端口：**

```bash
agentproxy http start --port 9090
```

或修改配置文件：

```properties
http.port=9090
```

**生成新 API Key：**

```bash
agentproxy http gen-key
```

**禁用 HTTP 服务：**

```properties
http.enabled=false
```

或启动时：

```bash
agentproxy --no-http
```

**配置定期清理：**

```properties
# 是否启用定期清理
cleanup.enabled=true

# 保留天数（清理超过此天数的历史消息）
cleanup.days=7

# 要清理的状态（逗号分隔，默认只清理已完成状态）
cleanup.status=DONE
```

清理规则：
- 每小时自动执行清理
- 仅清理指定状态的消息
- 不清理 PENDING、EXECUTING 等进行中的状态
```

### 6.3 数据存储位置

数据库：`~/.agentcommproxy/data.db`

日志：`~/.agentcommproxy/logs/`

## 八、常见问题

### 8.1 HTTP 服务无法启动

**问题：端口被占用**

```bash
# 检查端口占用
lsof -i :8080  # Linux
netstat -ano | findstr :8080  # Windows

# 解决：更换端口
agentproxy http start --port 9090
```

### 8.2 API Key 忘记

```bash
# 查看当前 API Key
agentproxy http status

# 或查看配置文件
cat ~/.agentcommproxy/config.properties | grep http.api.key
```

### 8.3 守护进程未运行

```bash
# 检查守护进程状态
agentproxy daemon status

# 手动启动
agentproxy daemon start --foreground
```

### 8.4 数据库损坏

```bash
# 备份后删除数据库
mv ~/.agentcommproxy/data.db ~/.agentcommproxy/data.db.bak

# 重新启动会自动创建新数据库
agentproxy daemon start --foreground
```

### 8.5 Windows 安装后命令无效

重新打开命令提示符窗口（PATH 变量需要刷新）。

## 九、升级说明

### 9.1 升级步骤

```bash
# 1. 停止服务
agentproxy daemon stop
agentproxy http stop

# 2. 备份配置和数据
cp ~/.agentcommproxy/config.properties ~/.agentcommproxy/config.properties.bak
cp ~/.agentcommproxy/data.db ~/.agentcommproxy/data.db.bak

# 3. 替换 JAR 文件
cp target/agentcommproxy-1.0.0.jar /opt/agentproxy/  # Linux
# 或 Windows: C:\Program Files\agentproxy\

# 4. 重新启动服务
agentproxy daemon start --foreground
```

## 十、卸载

### 10.1 Linux 卸载

```bash
# 停止服务
/opt/agentproxy/daemon.sh stop

# 删除安装目录
sudo rm -rf /opt/agentproxy
sudo rm -f /usr/local/bin/agentproxy

# 删除数据（可选）
rm -rf ~/.agentcommproxy

# 删除 systemd 服务（如已配置）
sudo systemctl disable agentproxy
sudo rm -f /etc/systemd/system/agentproxy.service
sudo systemctl daemon-reload
```

### 10.2 Windows 卸载

1. 停止运行中的服务
2. 删除 `C:\Program Files\agentproxy` 目录
3. 从系统 PATH 中移除该目录
4. 删除 `%USERPROFILE%\.agentcommproxy`（可选）
