# AgentCommProxy 项目介绍

## 一句话说明

AgentCommProxy 是一个 **Agent 消息代理服务**，让你的系统可以统一调用多种 AI Agent 工具（OpenClaw、Claude Code 等），无需关心每个工具的差异。

---

## 解决什么问题？

### 问题场景

团队使用多种 AI Agent 工具时，面临以下痛点：

| 问题 | 影响 |
|-----|------|
| 每个工具调用方式不同 | 需要分别学习 CLI 命令、API、参数 |
| 工具分散在本地运行 | 难以集中管理和监控 |
| 消息执行状态不透明 | 不知道任务是否完成、是否失败 |
| 回调通知难以统一 | 不同系统需要不同的通知方式 |
| 会话历史难以管理 | 不同任务的对话容易混在一起 |

### AgentCommProxy 的价值

**统一入口，简化调用**：
- 一套 API，调用所有 Agent 工具
- HTTP 接口 + CLI 命令，灵活选择
- 同步/异步两种模式，适配不同场景

**集中管理，状态透明**：
- 所有消息请求统一存储
- 实时查询执行状态
- 自动重试失败任务
- 支持回调通知或轮询查询

**会话隔离，任务独立**：
- MAIN 模式：共享默认会话
- INDEPENDENT 模式：每个任务独立对话上下文

## 核心能力

### 1. 多 Agent 工具支持

| 工具 | 说明 | 适用场景 |
|-----|------|---------|
| **OpenClaw** | OpenClaw CLI 命令执行 | 已有 OpenClaw 环境 |
| **Claude Code** | Claude Code CLI 命令执行 | 使用 Claude 进行代码分析、开发辅助 |

*接入新 Agent 工具只需实现接口，主系统无需改动*

### 2. 双入口模式

| 入口 | 适用场景 |
|-----|---------|
| **HTTP API** | 服务端集成、远程调用、系统对接 |
| **CLI 命令** | 本地测试、脚本调用、运维操作 |

### 3. 同步/异步执行

| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| **同步模式** | 发送后等待响应，立即返回结果 | 快速查询、简单任务 |
| **异步模式** | 发送后立即返回请求 ID，后台执行 | 长时间任务、批量处理 |

### 4. 多种回调方式

| 方式 | 说明 | 适用场景 |
|-----|------|---------|
| **HTTP 回调** | 任务完成后主动 POST 结果到指定地址 | 服务端对接 |
| **轮询查询** | 调用方主动查询状态接口 | 客户端/前端 |

### 5. 会话管理

| 模式 | 说明 |
|-----|------|
| **MAIN** | 默认模式，所有消息共享一个会话上下文 |
| **INDEPENDENT** | 每个业务任务 ID 有独立的对话历史，互不干扰 |

### 6. 可靠性保障

- 自动重试：执行失败/超时自动重试（可配置次数）
- 回调重试：回调失败自动重试
- 状态持久化：所有请求记录存 SQLite 数据库
- 守护进程：后台定时处理任务和重试

## 使用示例

### CLI 命令

```bash
# 发送消息（同步）
agentcommproxy agent --agent my-agent --sender test --message "hello" --sync

# 发送消息（异步）
agentcommproxy agent --agent my-agent --sender test --message "hello"

# 查询状态
agentcommproxy list --limit 10
agentcommproxy status <request-id>

# Claude Code 独立会话
agentcommproxy agent --agent my-claude-agent --sender test \
  --message "analyze this code" --proxy claude-code \
  --task-id "review-001" --session-mode independent --sync
```

### HTTP API

```bash
# 发送消息
curl -X POST http://localhost:8080/api/v1/send \
  -H "X-API-Key: your-key" \
  -H "Content-Type: application/json" \
  -d '{"agent":"my-agent","sender":"test","message":"hello","sync":true}'

# Claude Code 调用
curl -X POST http://localhost:8080/api/v1/send \
  -H "X-API-Key: your-key" \
  -H "Content-Type: application/json" \
  -d '{"agent":"my-claude-agent","sender":"test","message":"hello","sync":true,"proxy":"claude-code"}'

# 查询状态
curl http://localhost:8080/api/v1/status/<request-id> \
  -H "X-API-Key: your-key"
```

## 快速上手

### 1. 安装

```bash
# 编译打包
mvn package -DskipTests

# 启动 HTTP 服务
agentcommproxy http start --port 8080
```

### 2. 注册 Claude Code Agent

```bash
agentcommproxy claude-code register --agent my-agent --project /path/to/project
agentcommproxy claude-code list
```

### 3. 发送消息

```bash
# CLI
agentcommproxy agent --agent my-agent --sender test --message "hello" --sync

# HTTP
curl -X POST http://localhost:8080/api/v1/send \
  -H "X-API-Key: your-key" \
  -d '{"agent":"my-agent","sender":"test","message":"hello","sync":true}'
```

AgentCommProxy 让你的团队：

- ✅ 统一调用多种 Agent 工具，无需分别学习
- ✅ HTTP/CLI 双入口，灵活适配不同场景
- ✅ 同步/异步双模式，满足即时和后台需求
- ✅ 会话隔离，不同任务对话不混淆
- ✅ 自动重试，保障可靠性

- ✅ 易于扩展，接入新工具零改动

**一句话：AgentCommProxy 是你团队的 Agent 工具统一调度中心。**
