# 独立会话支持设计

## 背景

AgentCommProxy 需要支持独立会话模式，让不同任务有独立的对话上下文，同时支持多种 Agent 工具接入。

## 核心概念

### SessionMode

| 模式 | 说明 | sessionId 管理 |
|---|---|---|
| **MAIN** | 主会话模式，共享默认会话 | Claude Code: JSON配置；OpenClaw: 无此概念 |
| **INDEPENDENT** | 独立会话模式，每个 taskId 独立会话 | requests 表存储，taskId ↔ sessionId 映射 |

### 会话创建机制差异

| 工具 | 创建方式 | 一条命令能否完成 |
|---|---|---|
| **Claude Code** | `claude --session-id {uuid} -p "message"` | ✅ 创建+发送一体化 |
| **OpenClaw** | 需先编辑 sessions.json，再执行命令 | ❌ 需两步操作 |

## 架构设计

采用 AgentProvider 抽象层，封装各工具的会话管理差异：

```
┌─────────────────────────────────────────────────────────────┐
│                      AgentService                            │
│           只依赖 AgentProvider，不感知工具差异                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    AgentProvider (接口)                       │
│   getSessionManager() │ needsPreCreateSession() │ createIndependentSession() │
└─────────────────────────────────────────────────────────────┘
          │                    │
          ▼                    ▼
┌──────────────────┐ ┌──────────────────┐
│ OpenClawProvider  │ │ ClaudeCodeProvider │
│ needsPreCreate: true│ │ needsPreCreate: false│
└──────────────────┘ └──────────────────┘
```

### needsPreCreateSession 含义

不是"是否需要创建会话"，而是"是否需要在 execute **之前**单独创建会话"。

## 文件结构

### Provider 层

| 文件 | 职责 |
|---|---|
| `AgentProvider.java` | 接口定义 |
| `AgentProviderFactory.java` | 注册/获取 Provider |
| `OpenClawProvider.java` | OpenClaw 完整接入封装 |
| `ClaudeCodeProvider.java` | Claude Code 完整接入封装 |

### Session 层

| 文件 | 职责 |
|---|---|
| `SessionManager.java` | 接口，定义查询方法 |
| `BaseSessionManager.java` | 基类，封装 requests 表查询（通用） |
| `OpenClawSessionManager.java` | OpenClaw 实现（mainSessionId 返回 null） |
| `ClaudeCodeSessionManager.java` | Claude Code 实现（mainSessionId 存 JSON 配置） |
| `OpenClawSessionHelper.java` | sessions.json 操作（OpenClaw 专用） |

### Proxy 层

| 文件 | 职责 |
|---|---|
| `CommandProxy.java` | 接口，execute(agent, message, timeout, sessionId) |
| `CommandResult.java` | 结果封装，新增 sessionId 字段 |
| `OpenClawProxy.java` | OpenClaw 命令执行 |
| `ClaudeCodeProxy.java` | Claude Code 命令执行（--session-id/--resume） |

## 调用流程

### MAIN 模式

```
AgentService.sendSync()
  ↓
getSessionId() → sessionManager.getMainSessionId()
  ↓
【Claude Code】
  返回 JSON 配置中的 mainSessionId（可能为 null）
  ↓
  proxy.execute(..., sessionId)
    - null → --session-id {新UUID}
    - 有值 → --resume {sessionId}
  ↓
  返回 sessionId → 存入 JSON 配置

【OpenClaw】
  返回 null（无 mainSessionId 概念）
  ↓
  proxy.execute(..., null)
    → openclaw agent --agent {agent} ...
```

### INDEPENDENT 模式首次调用

```
AgentService.sendSync() / processExecute()
  ↓
getSessionId() → 返回 null（requests 表无记录）
  ↓
检查 provider.needsPreCreateSession()
  ↓
【Claude Code: false】
  直接 proxy.execute(..., null)
  Proxy 内部：生成 UUID + --session-id + 发送消息（一条命令）
  返回 CommandResult.sessionId → 存入 requests 表

【OpenClaw: true】
  先调用 provider.createIndependentSession()
    → OpenClawSessionHelper.ensureMainSessionExists()（发送 "hi"）
    → OpenClawSessionHelper.createIndependentSession()
      - 复制 main 会话配置
      - 生成 UUID
      - 写入 sessions.json
      - 创建空 .jsonl 文件
    → 返回 sessionId
  ↓
  再 proxy.execute(..., sessionId)
    → openclaw agent --session-id {sessionId} --message ...
  ↓
  返回结果 → 存入 requests 表
```

### INDEPENDENT 模式后续调用

```
getSessionId() → sessionManager.findSessionIdByTaskId()
  ↓
返回已有 sessionId（从 requests 表查询）
  ↓
【Claude Code】
  proxy.execute(..., sessionId)
    → claude --resume {sessionId} -p "message"

【OpenClaw】
  proxy.execute(..., sessionId)
    → openclaw agent --session-id {sessionId} --message ...
```

## 数据存储

### SQLiteStore requests 表

```sql
-- 新增字段
task_id TEXT,        -- 业务任务 ID（INDEPENDENT 模式）
session_id TEXT,     -- 底层会话 ID
```

### Claude Code 配置文件

**路径**: `~/.agentcommproxy/claude-code-agents.json`

```json
{
  "agents": [{
    "agentId": "my-agent",
    "projectPath": "/path/to/project",
    "mainSessionId": "uuid-xxx",  // MAIN 模式会话 ID
    "createdAt": "...",
    "updatedAt": "..."
  }]
}
```

### OpenClaw sessions.json

**路径**: `~/.openclaw/agents/{agentId}/sessions/sessions.json`

```json
{
  "agent:my-agent:main": {
    "sessionId": "uuid-main",
    "sessionFile": "~/.openclaw/agents/my-agent/sessions/uuid-main.jsonl"
  },
  "agent:my-agent:task-001": {
    "sessionId": "uuid-001",
    "sessionFile": "~/.openclaw/agents/my-agent/sessions/uuid-001.jsonl"
  }
}
```

## CLI 参数

```bash
--task-id <id>           # 业务任务 ID（INDEPENDENT 模式需要）
--session-mode <mode>    # main / independent，默认 main
--clear-session          # 清空会话历史
```

## HTTP API

```json
{
  "taskId": "order-123",
  "sessionMode": "independent",
  "clearSession": false
}
```

## 接入新 Agent 工具

参见 `07-AgentProvider扩展设计.md`

核心步骤：
1. 实现 `AgentProvider` 接口
2. 注册到 `AgentProviderFactory`
3. 主工程零改动

关键判断：`needsPreCreateSession` 值
- 工具支持命令行参数创建会话 → `false`
- 工具需要额外操作（配置文件/API）→ `true`

## 测试验证

```bash
# Claude Code INDEPENDENT 模式
agentcommproxy agent --agent my-agent --sender test \
  --message "hello" --task-id "task-001" \
  --session-mode independent --proxy claude-code --sync

# 验证会话复用
agentcommproxy agent --agent my-agent --sender test \
  --message "what was my previous question?" --task-id "task-001" \
  --session-mode independent --proxy claude-code --sync
```

预期：第二次调用能回答"hello"相关的历史问题。
