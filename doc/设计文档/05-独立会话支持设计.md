# 独立会话支持设计方案

## Context

**需求背景**: AgentCommProxy 目前每次调用 openclaw 都在同一个 main 会话中执行，需要支持独立会话模式，让不同任务有独立的对话上下文。

**关键约束**:
- 入口层不暴露底层 sessionId，只传递业务 taskId 和会话模式
- 从 main 会话复制创建新会话
- 无会话时先触发 main 创建

## 改造范围

| 层级 | 文件 | 改动 |
|------|------|------|
| 模型层 | AgentRequest.java | 新增 taskId、sessionMode、sessionId 字段 |
| 新增枚举 | SessionMode.java | MAIN / INDEPENDENT |
| 新增模块 | SessionManager.java | sessions.json 操作、taskId↔sessionId映射 |
| 代理层 | OpenClawProxy.java | buildCommand 增加 --session-id 参数 |
| 代理层 | CommandProxy.java | execute 方法签名增加 sessionId |
| HTTP DTO | SendRequest.java | 新增 taskId、sessionMode 字段 |
| CLI命令 | AgentCommand.java | 新增 --task-id、--session-mode 选项 |
| 存储层 | SQLiteStore.java | requests 表新增 task_id、session_id 字段 |
| 服务层 | AgentService.java | 根据 sessionMode 调用 SessionManager |

## 详细设计

### 1. SessionMode 枚举

```java
public enum SessionMode {
    MAIN,         // 主会话模式，使用默认 main 会话
    INDEPENDENT   // 独立会话模式，创建新会话
}
```

### 2. AgentRequest 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | String | 业务任务ID，由应用层提供 |
| sessionMode | SessionMode | 会话模式 |
| sessionId | String | 底层会话ID，系统内部管理 |
| clearSession | boolean | 是否清空会话历史，默认 false |

### 3. SessionManager 模块

**路径**: `src/main/java/org/openclaw/agentcommproxy/session/SessionManager.java`

**职责**:
- 管理 openclaw sessions.json 文件
- 创建独立会话（从 main 复制）
- 维护 taskId ↔ sessionId 映射
- 无会话时触发 main 创建

**核心方法**:

```java
public class SessionManager {
    // 获取或创建 sessionId
    // MAIN模式: 返回 "main" 或 main的sessionId
    // INDEPENDENT模式: 创建新会话，建立映射
    public String getOrCreateSessionId(String agentId, String taskId, SessionMode mode);

    // 从 main 复制创建新会话
    private String createIndependentSession(String agentId, String taskId);

    // 检查是否有会话，无则触发创建
    private void ensureMainSessionExists(String agentId);

    // 读取/写入 sessions.json
    private JsonObject readSessionsJson(String agentId);
    private void writeSessionsJson(String agentId, JsonObject sessions);
}
```

**sessions.json 路径**: `{openclawDataDir}/agents/{agentId}/sessions.json`

**配置项**:
- `openclaw.data.dir`: OpenClaw 数据目录，默认 `.openclaw`（即 `~/.openclaw`)
- 可配置绝对路径或相对路径，不配置时使用默认值

### 4. 创建会话流程

**关键约定**:
- openclaw `--session-id` 参数使用 sessionId（UUID格式）
- entry key 使用 taskId 命名：`agent:{agentId}:{taskId}`

```
INDEPENDENT 模式:

1. 检查 sessions.json 是否存在
   - 不存在 → 发送 "hi" 消息触发 main 创建

2. 从 sessions.json 读取 main 会话配置

3. 复制 main 配置，修改:
   - entry key: "agent:{agentId}:{taskId}"
   - sessionId: 生成新 UUID
   - updatedAt: 当前时间戳
   - sessionFile: "{sessionsDir}/{sessionId}.jsonl"

4. 写入 sessions.json

5. 创建空 .jsonl 文件（路径由 sessionFile 指定）

6. 建立映射: taskId → sessionId

7. 返回 sessionId（用于 openclaw --session-id 参数）
```

### 5. OpenClawProxy 改造

**CommandProxy 接口新增方法**:

```java
CommandResult execute(String agent, String message, int timeout, String sessionId);
```

**OpenClawProxy buildCommand**:

```java
public String buildCommand(String agent, String message, int timeout, String sessionId) {
    String escapedMessage = escapeMessage(message);
    if (sessionId != null && !sessionId.isEmpty()) {
        // 使用 --session-id 参数指定会话，session 已关联 agent，无需再指定
        return String.format("openclaw agent --session-id %s --message \"%s\" --timeout %d",
                sessionId, escapedMessage, timeout);
    } else {
        // 主会话模式，需要指定 agent
        return String.format("openclaw agent --agent %s --message \"%s\" --timeout %d",
                agent, escapedMessage, timeout);
    }
}
```

### 6. AgentService 改造

```java
public AgentRequest sendSync(AgentRequest request) {
    // 新增: 根据 sessionMode 获取 sessionId
    SessionManager sessionManager = new SessionManager(configManager);
    String sessionId = sessionManager.getOrCreateSessionId(
        request.getTargetAgent(),
        request.getTaskId(),
        request.getSessionMode()
    );
    request.setSessionId(sessionId);

    // ... 原有逻辑

    // execute 时传入 sessionId
    CommandResult result = proxy.execute(
        request.getTargetAgent(),
        request.getMessage(),
        request.getTimeout(),
        sessionId
    );
}
```

### 8. 入口层入参

**CLI AgentCommand**:

```bash
--task-id <id>           # 业务任务ID（可选）
--session-mode <mode>     # 会话模式 (main/independent)，默认 main
--clear-session <bool>   # 是否清空会话历史 (true/false)，默认 false，仅 independent 模式有效
```

**HTTP SendRequest**:

```json
{
  "taskId": "order-123",
  "sessionMode": "independent",
  "clearSession": false
}
```

**sessionMode 默认规则**:
- 未指定 `sessionMode`：默认使用 **MAIN**
- 指定 `sessionMode=main`：使用主会话
- 指定 `sessionMode=independent`：使用独立会话（需要 taskId）

**clearSession 参数说明**:
- `true`: 清空指定会话的聊天历史文件 (.jsonl)，会话从空白开始
- `false` (默认): 保留会话历史，新消息追加到现有历史中
- **仅 `sessionMode=independent` 时有效**
- `sessionMode=main` 时设置此参数会被忽略（MAIN 模式不支持清空）
