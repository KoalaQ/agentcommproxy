# 项目架构

## 整体架构

AgentCommProxy 采用分层架构设计，分为 CLI 层、HTTP 层、Service 层、Provider 层、Proxy 层和 Store 层。

```
┌─────────────────────────────────────────────────────────────┐
│                      CliRunner (入口)                        │
│                                                              │
│   ┌──────────────┐              ┌──────────────┐            │
│   │ CLI Handler  │              │ HTTP Handler │            │
│   │  (Picocli)   │              │ (HttpServer) │            │
│   └──────────────┘              └──────────────┘            │
│          │                              │                    │
│          └──────────────┬───────────────┘                    │
│                         │                                    │
│                         ▼                                    │
┌─────────────────────────────────────────────────────────────┐
│                      Service Layer                           │
│                     AgentService                             │
│         sendSync() │ sendAsync() │ processCallback()        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                     Provider Layer                           │
│              AgentProvider / AgentProviderFactory            │
│           封装 Agent 工具接入逻辑（会话管理 + 命令执行）         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      Proxy Layer                             │
│              CommandProxy / OpenClawProxy / ClaudeCodeProxy  │
│                  执行外部命令                                 │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      Store Layer                             │
│                     SQLiteStore                              │
│                  消息持久化存储                               │
└─────────────────────────────────────────────────────────────┘
```

## 核心模块说明

### 1. CLI Layer

入口类 `CliRunner` 使用 Picocli 框架实现命令行解析，包含以下子命令：

| 命令类 | 功能 | 文件位置 |
|-------|------|---------|
| AgentCommand | 发送消息（同步/异步） | `command/AgentCommand.java` |
| ListCommand | 查询消息列表 | `command/ListCommand.java` |
| StatusCommand | 查询消息状态 | `command/StatusCommand.java` |
| DaemonCommand | 守护进程管理 | `command/DaemonCommand.java` |
| HttpCommand | HTTP 服务管理 | `command/HttpCommand.java` |
| ClearCommand | 清理历史记录 | `command/ClearCommand.java` |

### 2. HTTP Layer

`HttpServerManager` 管理 HTTP API 服务，使用 JDK 内置 `HttpServer`：

- 端点 `/api/v1/send`: 发送消息（同步/异步）
- 端点 `/api/v1/status/{id}`: 查询消息状态
- 端点 `/api/v1/list`: 查询消息列表
- 端点 `/api/v1/health`: 健康检查

认证方式：请求头 `X-API-Key`

DTO 类：
- `SendRequest`: 发送请求
- `SendResponse`: 发送响应
- `StatusResponse`: 状态响应
- `ListResponse`: 列表响应
- `HealthResponse`: 健康检查响应
- `ErrorResponse`: 错误响应

### 3. Service Layer

`AgentService` 是核心业务逻辑类，提供以下核心方法：

- **sendSync()**: 同步发送消息，立即执行并等待响应
- **sendAsync()**: 异步发送消息，保存请求后返回
- **processExecute()**: 处理异步消息执行（由 DaemonManager 调用）
- **processCallback()**: 处理异步消息回调（使用策略模式）
- **handleCallbackFailure()**: 处理回调失败，管理重试逻辑
- **handleExecuteFailure()**: 处理执行失败，管理重试逻辑

### 4. Callback Layer

回调层使用策略模式处理不同类型的回调：

```
┌─────────────────────────────────────────────────────────────┐
│                    CallbackHandlerFactory                    │
│                  (工厂类，根据 SenderType 创建 Handler)        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    CallbackHandler (接口)                     │
│                       doCallback(request)                    │
└─────────────────────────────────────────────────────────────┘
          │                    │                    │
          ▼                    ▼                    ▼
┌──────────────────┐ ┌──────────────────┐ ┌──────────────────┐
│  CliCallbackHandler │ │HttpCallbackHandler │ │ NoopCallbackHandler │
│   (CLI 回调实现)    │ │ (HTTP 回调实现)    │ │  (轮询模式，无回调)  │
└──────────────────┘ └──────────────────┘ └──────────────────┘
```

**SenderType 类型：**

| 类型 | 说明 | 回调方式 |
|-----|------|---------|
| CLI | CLI 命令发起 | 执行 proxy.execute() |
| HTTP_CALLBACK | HTTP + callbackUrl | POST 回调地址 |
| HTTP_POLL | HTTP 无 callbackUrl | 无回调，调用方轮询 |

### 5. Provider Layer

Provider 层封装 Agent 工具的完整接入逻辑，是 Service 层与 Proxy 层之间的桥梁：

```
┌─────────────────────────────────────────────────────────────┐
│                    AgentProviderFactory                       │
│                  (工厂类，注册和获取 AgentProvider)            │
└─────────────────────────────────────────────────────────────┐
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    AgentProvider (接口)                       │
│   getSessionManager() │ getProxy() │ createIndependentSession() │
└─────────────────────────────────────────────────────────────┘
          │                    │
          ▼                    ▼
┌──────────────────┐ ┌──────────────────┐
│ OpenClawProvider  │ │ ClaudeCodeProvider │
│ (OpenClaw 接入)   │ │ (Claude Code 接入) │
└──────────────────┘ └──────────────────┘
```

**核心职责：**

| 方法 | 说明 |
|-----|------|
| `getSessionManager()` | 获取会话管理器（MAIN/INDEPENDENT 模式） |
| `getProxy()` | 获取命令代理 |
| `needsPreCreateSession()` | 是否需要在 execute 前预创建会话 |
| `createIndependentSession()` | 创建独立会话（INDEPENDENT 模式首次调用） |

**会话创建机制差异：**

| 工具 | needsPreCreateSession | 创建方式 |
|-----|----------------------|---------|
| Claude Code | false | `--session-id` 参数，一条命令完成创建+发送 |
| OpenClaw | true | 需先编辑 sessions.json，再执行命令 |

**接入新 Agent 工具只需：**

1. 实现 `AgentProvider` 接口
2. 注册到 `AgentProviderFactory`
3. 主工程零改动

### 6. Proxy Layer

代理层负责与外部 Agent 系统通信，执行具体命令：

```
┌─────────────────────────────────────────────────────────────┐
│                    CommandProxy (接口)                        │
│                       execute(agent, message, timeout, sessionId) │
└─────────────────────────────────────────────────────────────┘
          │                    │                    │
          ▼                    ▼                    ▼
┌──────────────────┐ ┌──────────────────┐ ┌──────────────────┐
│  OpenClawProxy    │ │ ClaudeCodeProxy   │ │  其他 Proxy...   │
│  (OpenClaw CLI)   │ │ (Claude Code CLI) │ │                  │
└──────────────────┘ └──────────────────┘ └──────────────────┘
```

**核心组件：**

- **CommandProxy**: 接口定义，定义 `execute()` 方法
- **CommandResult**: 执行结果封装（包含 output、error、sessionId）
- **OpenClawProxy**: OpenClaw 系统实现
- **ClaudeCodeProxy**: Claude Code CLI 实现
- **CommandResult**: 执行结果封装（包含输出、错误、超时状态）

### 6. Daemon Layer

`DaemonManager` 单例类管理后台守护线程：

- 使用 `ScheduledExecutorService` 定时扫描待处理消息
- 使用 `ExecutorService` 线程池执行消息处理任务
- 支持前台模式（阻塞）和后台模式（非阻塞）
- 处理新请求和重试请求

### 6. Store Layer

`SQLiteStore` 负责消息持久化：

- 数据库位置: `~/.agentcommproxy/messages.db`
- 表结构: `requests` 主表 + `retry_queue` 重试队列
- 提供请求的增删改查和状态更新

### 7. Config Layer

`ConfigManager` 管理配置：

- 配置文件位置: `~/.agentcommproxy/config.properties`
- 支持默认超时、重试次数、重试间隔、线程池大小等配置
- 支持 HTTP 服务配置（端口、启用状态、API Key）

## 消息状态流转

### 同步模式

```
PENDING → EXECUTING → DONE/ERROR
```

同步模式简单直接：保存请求后立即执行，根据执行结果设置为 DONE 或 ERROR。

### 异步模式

```
PENDING → EXECUTING → EXECUTE_SUCCESS → CALLBACKING → DONE
                   ↘ EXECUTE_FAILED   ↘ CALLBACK_FAILED
                   ↘ EXECUTE_TIMEOUT
                   ↘ ERROR (重试耗尽)
```

异步模式流程：

1. **PENDING**: 请求已保存，等待守护进程处理
2. **EXECUTING**: 守护线程正在执行
3. **EXECUTE_SUCCESS**: 执行成功，准备回调
4. **CALLBACKING**: 正在回调发送方
5. **DONE**: 回调成功，流程完成

失败路径：

- **EXECUTE_FAILED**: 执行失败，进入重试队列
- **EXECUTE_TIMEOUT**: 执行超时，进入重试队列
- **CALLBACK_FAILED**: 回调失败，进入重试队列
- **ERROR**: 重试次数耗尽后的最终失败状态

## 重试机制

系统支持两种类型的重试：

1. **执行重试**: `EXECUTE_FAILED` / `EXECUTE_TIMEOUT` 状态的消息
2. **回调重试**: `CALLBACK_FAILED` 状态的消息

重试逻辑：

- 每次失败后将请求加入 `retry_queue` 表
- 记录下次重试时间（当前时间 + 重试间隔）
- 守护线程定时扫描重试队列
- 达到最大重试次数后标记为 ERROR

## 数据模型

### AgentRequest

核心请求实体，包含字段：

| 字段 | 类型 | 说明 |
|-----|------|-----|
| id | String | UUID 请求唯一标识 |
| sender | String | 发送方 Agent 名称 |
| targetAgent | String | 目标 Agent 名称 |
| message | String | 消息内容 |
| response | String | 响应内容 |
| error | String | 错误信息 |
| status | MessageStatus | 消息状态 |
| sync | boolean | 是否同步模式 |
| timeout | int | 超时时间（秒） |
| executeRetryCount | int | 执行重试次数 |
| callbackRetryCount | int | 回调重试次数 |
| senderType | SenderType | 请求方类型 |
| callbackUrl | String | HTTP 回调地址 |
| proxyType | ProxyType | Proxy 类型 |

### SenderType

请求方类型枚举：

```java
CLI              // CLI 命令发起，使用 CLI 回调
HTTP_CALLBACK    // HTTP API 发起，提供回调地址
HTTP_POLL        // HTTP API 发起，轮询模式
```

### ProxyType

Proxy 类型枚举：

```java
OPENCLAW         // OpenClaw CLI 命令执行
HTTP             // HTTP API 调用
WEBSOCKET        // WebSocket 通信
CUSTOM           // 自定义命令
```

### MessageStatus

状态枚举，包含所有可能的消息状态：

```java
PENDING          // 待处理
EXECUTING        // 正在执行
EXECUTE_SUCCESS  // 执行成功
EXECUTE_FAILED   // 执行失败
EXECUTE_TIMEOUT  // 执行超时
CALLBACKING      // 正在回调
CALLBACK_FAILED  // 回调失败
DONE             // 完成
ERROR            // 错误（最终失败）
```
